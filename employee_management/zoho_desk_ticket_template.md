## Ticket Workflow

### Stage 1: Initiated (Open)
- Off-boarding ticket created
- Reports generated
- Stakeholders notified

### Stage 2: Planning (In Progress)
- Reassignment plan approved
- Transition timeline established
- Knowledge transfer scheduled

### Stage 3: Execution (In Progress)
- Access revocation in progress
- Tickets being reassigned
- Projects being transferred
- Customers being notified

### Stage 4: Monitoring (Pending)
- Access fully revoked
- All assignments transferred
- Customer transitions in progress

### Stage 5: Completed (Closed)
- All access confirmed revoked
- All assignments completed
- All customers transitioned
- Exit interview completed
- Assets returned
- Final audit passed

## Automated Actions (Configure in Zoho Desk)

### On Ticket Creation
1. **Notify IT Manager:** Email notification
2. **Notify HR:** Email notification
3. **Create Subtasks:**
   - Generate Access Revocation Checklist
   - Generate Ticket Report
   - Generate Project Report
   - Generate Customer Report (if helpdesk role)
   - Schedule Exit Interview

### On Last Working Day
1. **Trigger:** Date field = Today
2. **Action:** Send reminder to execute final access revocation

### On Ticket Close
1. **Require:** All subtasks completed
2. **Action:** Send completion summary to stakeholders

## Integration with Automation Script

When creating off-boarding ticket in Zoho Desk:

1. Create ticket using template above
2. Run automation script:
   ```bash
   python offboarding_automation.py \
     --employee [EMPLOYEE_ID] \
     --name "[EMPLOYEE_NAME]" \
     --email [EMAIL] \
     --role [ROLE]
   ```
3. Attach generated reports to ticket
4. Update custom fields with data from reports
5. Create subtasks based on role

5. **Set Up Scheduled Reports** (Reports > Create Custom Report)
6. **Test Template** with sample off-boarding
